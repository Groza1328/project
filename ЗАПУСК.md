# Инструкция по запуску приложения СибМобиль

## Что это за приложение?

Это **Spring Boot веб-приложение** для аренды автомобилей. Оно работает через:
- **Spring Boot 3.4.12** - основной фреймворк
- **Java 17** - язык программирования
- **PostgreSQL** - база данных
- **Thymeleaf** - шаблонизатор для HTML страниц
- **Spring Security** - безопасность и аутентификация
- **Spring Mail** - отправка email уведомлений

## Предварительные требования

✅ **Java 17** - установлена (версия 17.0.17)
✅ **Maven** - доступен через Maven Wrapper

❓ **PostgreSQL** - нужно проверить, запущен ли сервер

## Шаги для запуска

### 1. Убедитесь, что PostgreSQL запущен

В `application.properties` настроено подключение к базе данных:
- База данных: `sibmobile`
- Порт: `5432`
- Пользователь: `postgres`
- Пароль: `Groza_13_28`

**Важно:** Если база данных `sibmobile` не существует, Spring Boot создаст её автоматически (благодаря настройке `spring.jpa.hibernate.ddl-auto=update`).

### 2. Запуск приложения

Есть два способа запуска:

#### Способ 1: Через Maven Wrapper (рекомендуется)
```powershell
cd c:\Users\Home\test\sibmobile
.\mvnw.cmd spring-boot:run
```

#### Способ 2: Через скомпилированный JAR
```powershell
cd c:\Users\Home\test\sibmobile
.\mvnw.cmd clean package
java -jar target\sibmobile-0.0.1-SNAPSHOT.jar
```

### 3. Доступ к приложению

После запуска приложение будет доступно по адресу:
- **http://localhost:8080**

Порт можно изменить в `application.properties` (сейчас: `server.port=8080`)

## Основные страницы приложения

- `/` или `/login` - страница входа
- `/register` - регистрация
- `/verify` - подтверждение email
- `/home` - главная страница (требует авторизации)
- `/fleet` - автопарк (требует авторизации)
- `/order` - оформление заказа (требует авторизации)
- `/tariffs` - тарифы (требует авторизации)


## Остановка приложения

Нажмите `Ctrl + C` в терминале, где запущено приложение.

## Режим разработки

Приложение использует Spring Boot DevTools, что означает:
- Автоматическая перезагрузка при изменении кода
- Отключён кеш Thymeleaf для удобства разработки

## Проблемы при запуске?

1. **Ошибка подключения к PostgreSQL:**
   - Убедитесь, что PostgreSQL запущен
   - Проверьте настройки в `application.properties`

2. **Порт 8080 занят:**
   - Измените `server.port` в `application.properties`
   - Или остановите процесс, использующий порт 8080

3. **Ошибки компиляции:**
   - Запустите `.\mvnw.cmd clean install` для очистки и пересборки
